# role
第１１回メンバー役割

1. リーダー (PM & ドキュメント)

【役割】プロジェクト全体の進行管理と、最終成果物の仕上げ（README）。

【担当タスク】
マージ作業: メンバーからのPull Requestを確認し、コンフリクトを解消してmainブランチに統合する。

2. メンバーA (バックエンド: 集計ロジック担当)

【役割】「データの集計・統計」を行うデータベース処理の実装。

【担当タスク】
集計クエリの作成: app.py またはモデルファイルにて、以下のデータを抽出する処理を書く。

スタッフごとのシフト希望提出回数(今月)

時間帯ごとの人気倍率(今月)

男女比率

3. メンバーB (フロントエンド: ダッシュボード担当)

【役割】メンバーAが抽出したデータを、トップページで見やすく表示する。

【担当タスク】
可視化: 集計データを「ランキング表」や、可能なら「グラフ（Chart.jsなどのライブラリ使用）」で表示する。
デザイン調整: 重要な数字を大きく表示するなど、CSSで見やすく整える。

4. メンバーC (機能拡張: マスターデータ管理担当)

【役割】「スタッフ」や「シフト枠」の情報を後から修正・削除できるようにする。

【担当タスク】
編集機能: スタッフ名の修正や、間違えて登録したシフト枠の日時変更機能を実装する。
削除機能: 辞めたスタッフを削除する機能を実装する。
画面改修:「編集」「削除」ボタンを追加する。

5. メンバーD (機能拡張: トランザクション管理担当)

【役割】「シフト希望」の取り消しの実装。

【担当タスク】
取り消し機能: 一度出したシフト希望を取り下げる（削除する）機能を実装する。
重複チェック: 「同じ人が同じ日時に2回希望を出せないようにする」チェック処理を入れる。
UI改善: 希望提出画面で、すでに希望済みの枠には「提出済」と表示するなど、使い勝手を向上させる。
